# Security Policy

## Overview

The PapsAI XNet project is committed to developing secure, reliable, and reproducible research software for explainable artificial intelligence in cervical cancer screening.

We appreciate responsible disclosure of security vulnerabilities and will make every effort to investigate and address legitimate security concerns in a timely manner.

PapsAI XNet is currently released as **research software** and is **not intended for direct clinical diagnosis or patient management** without appropriate regulatory approval and clinical validation.

---

# Reporting a Security Vulnerability

If you discover a security vulnerability, **please do not create a public GitHub Issue**.

Instead, report the vulnerability privately by contacting the project maintainers.

- Potential impact

- Suggested mitigation (if available)

---

# What to Expect

After receiving a vulnerability report, we will:

1. Acknowledge receipt within **5 business days**.
2. Investigate the reported issue.
3. Determine the severity and potential impact.
4. Develop and test an appropriate fix.
5. Release a security update if necessary.
6. Credit the reporter where appropriate (unless anonymity is requested).

---

# Responsible Disclosure

We kindly ask security researchers to:

- Give us reasonable time to investigate and resolve vulnerabilities before public disclosure.
- Avoid accessing, modifying, or deleting data belonging to others.
- Avoid actions that could disrupt users or infrastructure.
- Act in good faith throughout the disclosure process.

---

# Scope

This policy applies to:

- Source code
- Docker images
- Configuration files
- Build scripts
- Python packages
- Documentation
- GitHub repository

It does **not** apply to:

- Third-party dependencies
- External datasets
- Operating system vulnerabilities
- Cloud infrastructure managed by third parties

---

# Dependency Security

We periodically review project dependencies to reduce security risks.

Security practices include:

- Updating dependencies regularly.
- Removing deprecated packages.
- Reviewing dependency vulnerabilities.
- Monitoring GitHub Dependabot alerts.
- Applying security patches where appropriate.

---

# Research Software Disclaimer

PapsAI XNet is released to support **research, education, transparency, and reproducibility**.

The software has **not** been approved by any regulatory authority as a clinical diagnostic device.

Results generated by PapsAI XNet should **not** be used as the sole basis for clinical decision-making.

Any clinical deployment requires:

- Independent validation
- Regulatory approval
- Appropriate clinical governance
- Human expert oversight

---

# Future Security

Future releases, including **PapsAI XNet V2** and components integrated into the **PapsAI Digital Microscopy Ecosystem**, will continue to strengthen software security, robustness, privacy, and compliance with emerging best practices for trustworthy AI in healthcare.
